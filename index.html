GIT Y GITHUB

Git es un sistema de control de revisiones distribuido, rápido y escalable con un conjunto de comandos inusualmente rico que proporciona operaciones de alto nivel y acceso completo a los componentes internos.

1.-Git

Git es un sistema de control de versiones, lo que eso significa realmente es que Git nos ayuda a gestionar nuestros archivos de proyecto. Una de las cosas principales que hace Git y también la razón principal por la que existe es para mantener un registro de toda la historia de las cosas en las que estás trabajando.


2.-GitHub

GitHub es un servicio basado en la web para el control de versiones usando Git. Básicamente, es un sitio de redes sociales para desarrolladores. Puedes mirar el código de otras personas, identificar los problemas con tu código e incluso proponer cambios. Esto también te ayuda a mejorar tu código. En una nota más ligera, es un gran lugar para mostrar tus proyectos y ser notado por los reclutadores potenciales


3.-Comandos Basicos de Git

  • Inicializar un repositorio Git

    • Este comando convierte tu carpeta en un repositorio de Git: git init .

  • Ver el estado del repositorio

    • Muestra los archivos modificados, añadidos, y el estado general del repositorio: git status

  • Agregar archivos al área de preparación (staging)

    • Agrega un archivo específico al área de staging (preparación) para que esté listo para el siguiente commit. Puedes usar . en lugar de un nombre de archivo para agregar todos los archivos: git add nombre-del-archivo

  • Confirmar los cambios (commit)

    • Guarda los cambios en el historial de Git con un mensaje que describe los cambios: git commit -m "Mensaje descriptivo de los cambios"

  • Ver el historial de commits

    • Muestra un historial de todos los commits realizados: git log


3.-Comandos Basicos de GitHub

  • Crear un repositorio en GitHub

    • Ve a github.com y crea un nuevo repositorio (haz clic en "New" y dale un nombre). GitHub te mostrará instrucciones para conectar tu repositorio local a este repositorio en la nube.

  • Conectar tu repositorio local al repositorio remoto en GitHub

    • Copia la URL del repositorio de GitHub (por ejemplo, https://github.com/tuusuario/nombre-del-repositorio.git) y usa el siguiente comando en tu terminal: git remote add origin https://github.com/tuusuario/nombre-del-repositorio.git

  • Subir los cambios al repositorio de GitHub

    • Si es la primera vez que subes tu proyecto, usa el siguiente comando para enviar todos los commits al repositorio remoto: git push -u origin main

  • Después de esta primera vez, puedes simplemente usar git push para subir cambios futuros.

4.-GitHub Pages

GitHub Pages es un servicio gratuito de GitHub que permite a los usuarios crear y alojar sitios web estáticos directamente desde un repositorio de GitHub. Esto es útil para presentar proyectos, crear documentación o incluso alojar un sitio web personal sin necesidad de configurar un servidor propio.

  • Alojamiento Gratuito

    • GitHub Pages permite alojar un sitio web estático sin costo, lo cual es ideal para sitios personales, portafolios, blogs, o documentación de proyectos.

  • Compatible con Jekyll

    • GitHub Pages es compatible con Jekyll, un generador de sitios estáticos. Con Jekyll, puedes crear sitios más personalizados usando plantillas y escribir contenido en Markdown, que luego se convierte a HTML automáticamente.

  • Dominio Personalizado

    • Puedes usar un dominio personalizado (por ejemplo, tudominio.com) para tu sitio en GitHub Pages.
